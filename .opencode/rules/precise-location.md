# 精准定位规则 (PRECISE LOCATION RULES)

## 🚨 黄金法则
**禁止在完成定位前扫描代码。地毯式搜索是最后手段。**

## 定位流程

### 免定位（Level 0）：满足任一即跳过三步
- 单文件修改（配置/样式/常量）
- 异常堆栈已明确指向具体文件
- 纯文本/文档修改

### 标准定位（Level 1-3）：按序执行
| 步骤 | 动作 | 产出 |
|------|------|------|
| 1.模块定位 | 查阅 doc/detailed/ 确定业务模块 | 模块名 |
| 2.层级定位 | 视图/业务/数据/接口/配置 | 层级名 |
| 3.文件定位 | 推断 1~3 个候选文件，并行读取 | 文件路径 |

### 输出格式（定位声明）
```
📍 模块 → 层级 → 候选文件
```

## 例外（允许扩大搜索，须说明原因）
| 场景 | 动作 |
|------|------|
| 详设无对应模块 | 说明后扩大 |
| 候选文件无相关逻辑 | 说明后扩大至同层 |
| 跨模块全局Bug | 说明后逐一排查 |

## 禁止
- ❌ 不经定位直接全局扫描
- ❌ 定位到目标后仍读无关文件
- ❌ 有详设却跳过不读
