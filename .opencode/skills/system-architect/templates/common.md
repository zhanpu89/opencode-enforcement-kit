# SAD 文档通用骨架

> 通用骨架（25 章 + 头部 + 尾部），所有端类型共享。
> 端专属章节（前端 F1-F6 / 小程序 M1-M6）见 `templates/end-specific.md`。

---

## 文档头部格式

```
# {项目名} — 系统架构设计文档

**文档编号**：SAD-YYYYMMDD-NNN
**版本**：v1.0.0
**状态**：🟡 草稿
**创建日期**：YYYY-MM-DD
**最后更新**：YYYY-MM-DD
**作者**：[姓名]
**审核人**：[姓名 / 待定]
**关联文档**：{PRD文档路径（PRD-YYYYMMDD-NNN）/ 无}
```

---

## 文档类型与章节说明

> - **后端文档**（`{项目名}_SAD_后端.md` 或纯后端 `{项目名}_SAD.md`）：包含全部章节（1-25 + 变更记录）。
> - **前端文档**（`{项目名}_SAD_前端.md`）：包含章节 1-6、F1-F6、22-25，跳过后端专属章节（7-21）。
> - **小程序文档**（`{项目名}_SAD_小程序.md`）：包含章节 1-6、M1-M6、22-25，跳过后端专属章节（7-21）。
> - **概览文档**（`{项目名}_SAD_概览.md`）：仅包含章节 1-3 和端类型声明，500 字以内。
>
> 前端/小程序专属章节（F1-F6、M1-M6）→ 参见本文件末尾 **PART 1**（Web 前端）和 **PART 2**（微信小程序）

## 章节详细定义

### 1. 文档头部

见上方"文档头部格式"。

### 2. 目录

根据文档中实际存在的章节自动生成。

### 3. 执行摘要

- 系统概述（一句话描述系统做什么）
- 核心架构特点（2-3 个关键设计决策）
- 技术栈概览
- 关键约束与风险

### 4. 需求分析

- 功能需求总结（来自 PRD，不重复写，引用即可）
- **非功能需求（必须量化）**：
  - 并发：峰值 X 并发用户
  - 响应时间：P99 ≤ Xms，P95 ≤ Xms
  - 可用性：X%（年停机 ≤ X 小时）
  - 数据保留：X 年
- 约束条件（技术栈限制、合规要求、预算）

### 5. 系统架构

- 高层架构图（ASCII 图或分层文字描述）
- 组件说明（每个组件的职责）
- 服务边界（哪些是独立服务，哪些是模块）
- 数据流（核心业务流程的数据流向）
- 集成点（外部系统、第三方 API）

### 6. 技术栈

每个关键组件必须包含：
- **是什么**：简要描述
- **为什么选它**：关键原因（参照 `resources/tech-selection.md` 技术选型 6 个评估维度）
- **为什么不选其他**：1-2 个备选方案及排除原因
- **已知权衡**：接受的弱点

### 7. 数据库设计

> SAD 层粒度：核心表名 + 关键字段 + ER 关系 + 索引策略概述。完整 DDL 由 task-decomposer 生成。

- 数据库选型说明
- 核心表清单（表名 + 用途 + 关键字段）
- ER 关系描述（主要关联关系）
- 索引策略（高频查询字段）
- 分区/分表策略（如需要）

### 8. API 设计

> SAD 层粒度：端点列表 + HTTP 方法 + 功能描述 + 认证方案。完整 OpenAPI Schema 由 task-decomposer 生成。

- API 架构（RESTful / GraphQL / gRPC）
- 主要端点列表（路径 + 方法 + 功能描述）
- 认证和授权方案
- 限流策略
- 错误处理规范

### 9. 模块设计

> SAD 层粒度：模块划分 + 职责描述 + 依赖关系。伪代码逻辑由 task-decomposer 生成。

- 核心模块划分
- 每个模块的职责（一句话）
- 模块间依赖关系图
- 关键接口定义（接口名 + 入参 + 出参，不含实现细节）

### 10. 特殊集成设计

> 仅当系统涉及区块链/支付/文件存储时包含。详细规范参见 `resources/db-security-integration.md` 特殊集成节。

**区块链集成**：上链策略、智能合约代码、哈希计算规范、异步上链流程、节点规划、链上验证流程。

**支付集成**：支付流程时序图（含回调）、幂等性策略、退款流程、签名验证。

**文件存储集成**：Bucket 结构、直传 vs 服务端转发决策、预签名 URL TTL 策略、文件哈希计算。

### 11. 缓存策略

- 缓存层选型（Redis / 本地缓存）
- 缓存键设计规范
- 缓存失效策略（TTL / 主动失效）
- 热数据识别

### 12. 消息队列设计

- 消息队列选型
- 主要消息类型和 Topic 设计
- 消费者组设计
- 错误处理和重试策略

### 13. 安全设计

> 逐项覆盖 `resources/db-security-integration.md` 安全检查清单 + 对应语言 Overlay 的安全节。

- 认证方案（Token 机制、存储、吊销）
- 授权模型（RBAC 三表、端点权限声明）
- 敏感数据加密（密码哈希、PII 字段加密 + 哈希列）
- API 安全（限流、防重放、输入校验）
- 审计日志（写操作记录、保留期）
- 合规（适用法规）

### 14. 性能优化

- 数据库优化（索引、查询优化、读写分离）
- 缓存策略（热数据、缓存穿透防护）
- 异步处理（哪些操作异步化）
- CDN 使用（静态资源）

### 15. 可扩展性设计

- 水平扩展策略（无状态设计、Session 外置）
- 垂直扩展上限
- 数据库扩展（读写分离、分库分表触发条件）
- 服务拆分路径（未来演进方向）

### 16. 监控和告警

- 关键指标（QPS、P99 延迟、错误率、CPU/内存）
- 日志策略（结构化日志、日志级别规范）
- 告警规则（阈值 + 通知方式）
- 监控工具选型

### 17. 部署架构

- 环境划分（开发/测试/生产）
- 容器化策略（Docker + docker-compose / K8s）
- CI/CD 流程概述
- 配置管理（环境变量、Secret 管理）

### 18. 灾难恢复

- 备份策略（频率、保留期、存储位置）
- RTO（恢复时间目标）
- RPO（恢复点目标）
- 故障转移策略

### 19. 成本分析

- 基础设施成本估算（按规模）
- 开源 vs 商业方案成本对比
- 成本优化建议

### 20. 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| 技术风险 | 高/中/低 | 高/中/低 | ... |

### 21. 实现路线图

- MVP 阶段（必须有的功能）
- 第二阶段（核心功能完善）
- 第三阶段（优化和扩展）
- 时间估计

### 22. 开发指南

- 项目结构（参照 `resources/overlays.md` 对应语言的分层模型）
- 编码规范要点（参照 `resources/overlays.md` 对应语言的编码规范）
- 本地开发环境搭建步骤

### 23. 测试策略

- 单元测试覆盖率目标
- 集成测试关键场景
- 性能测试基准
- 安全测试要点

### 24. 名词解释与缩写

定义文档中使用的所有技术术语和缩写。

### 25. 附录

相关文档链接、参考资料。

## 版本号变更规则

| 段位 | 触发条件 |
|------|---------|
| MAJOR | 架构方向根本性变化、文档推翻重写 |
| MINOR | 新增/删除章节，技术选型调整 |
| PATCH | 局部修正、评审意见小改、错别字 |

**变更类型标记：** 🆕 新建 → v1.0.0 | 🐛 修正 → PATCH+1 | ✏️ 修改 → PATCH+1 | ➕ 新增 → MINOR+1 | 🗑️ 删除 → MINOR+1 | 🔄 重构 → MAJOR+1

**强绑定规则：** 头部"版本"= 变更记录最后一行版本；头部"最后更新"= 变更记录最后一行日期。
