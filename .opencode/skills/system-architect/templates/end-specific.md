### F2. 页面路由设计

列出核心页面清单，包含路由路径、对应功能和权限要求：

| 页面名称 | 路由路径 | 功能说明 | 权限要求 |
|---------|---------|---------|---------|
| 登录页 | `/login` | 用户登录/注册 | 公开 |
| 首页 | `/` | 系统主入口 | 已登录 |
| 详情页 | `/detail/:id` | 单条记录详情 | 已登录 |

- **路由守卫**：未登录跳转 `/login`，无权限跳转 403 页面
- **路由懒加载**：非首屏页面使用动态 `import()` 懒加载

---

### F3. 状态管理方案

- **全局状态划分**：用户信息（userStore）、业务数据（各业务 Store）、UI 状态（uiStore）
- **Store 模块划分**：列出各 Store 名称及职责
- **持久化策略**：哪些状态需要持久化（localStorage / sessionStorage）

---

### F4. 组件拆分策略

- **公共组件清单**：布局组件（Header/Sidebar/Footer）、表单组件、表格组件、弹窗组件等
- **业务组件**：各业务模块的专属组件
- **组件通信规范**：Props / Emit / Provide-Inject / Store 的使用边界

---

### F5. 前后端契约

- **唯一事实来源**：以后端 OpenAPI 规范为唯一事实来源，前端 API 层 URL 必须与后端定义严格对应
- **API 层封装**：统一的请求拦截器（Token 注入）、响应拦截器（错误处理、Token 刷新）
- **类型定义**：从 OpenAPI 生成或手动维护 TypeScript 类型定义文件

---

### F6. 前端安全与性能

**安全：**
- Token 存储方式（localStorage / sessionStorage / httpOnly Cookie）及选型理由
- 路由守卫实现（未登录跳转、角色权限校验）
- 敏感操作二次确认（删除、支付等）

**性能：**
- 路由懒加载策略（哪些页面懒加载）
- 首屏优化（骨架屏、预加载）
- 静态资源 CDN 策略

- 以后端 OpenAPI 为唯一事实来源，请求 URL 与后端定义严格对应
