# 审计系统

## 执行流程

### Step 1：运行审计脚本

```bash
bash scripts/gate.sh audit
```

### Step 2：解读输出

根据审计输出，逐阶段分析异常项：

| 异常标记 | 含义 | 建议 |
|---------|------|------|
| ⚠️ 门禁已通过但无评审报告 | 可能为伪造 pass | 建议撤销门禁，补评审 |
| ❌ 评审不通过 | 存在 P0 阻断 | 修复后重评 |
| ⚠️ 评审结论通过但存在 P0 | 报告内容与结论不一致 | 人工检查评审报告 |
| ⏳ 缺文档 | 对应阶段无产出物 | 加载对应 skill 生成 |
| ⏳ 缺评审 | 文档已有但未评审 | 加载 review-expert 评审 |

### Step 3：询问用户处理

输出审计结论后，询问用户是否要处理异常项：

```
审计完成，发现 N 个异常：
1. [arch] 门禁通过但无评审报告 → 是否撤销门禁？
2. [detailed] 门禁通过但无评审报告 → 是否撤销门禁？
3. [prd] 评审有 3 个 P0 未修复 → 是否需要修复？
```

根据用户答复执行对应操作（`gate.sh unpass`、加载对应 skill 等）。
