# 评审报告模板集

本文件包含4种评审报告模板 + 全流程追溯矩阵模板（共5 `##` 节）。生成报告时，根据评审模式跳转对应 `## 模板X` 节，填充所有占位符，不留空白项。

> ⛔ **使用强制规则**：
> - 最终输出**必须严格遵循**对应模板的完整结构，不得增删顶层章节（一~五 + 变更记录）
> - 所有 `{...}` 占位符**必须替换**为项目实际内容，禁止保留任何占位符
> - 模板中的 `✅/❌`、`P{0/1/2}`、`是/否` 等选项**必须选定一个**，不得保留多选形式
> - 报告的"就绪检查结果"一节引用 `resources/check-common.md` → 「零、就绪检查清单」

## 版本号变更规则（报告类文档）

格式：`vMAJOR.MINOR.PATCH`

| 段位 | 触发条件 | 示例 |
|------|---------|------|
| **MAJOR** | 评审结论根本性变化（如从通过变为不通过） | v1.0.0 → v2.0.0 |
| **MINOR** | 新增评审维度或重大补充 | v1.0.0 → v1.1.0 |
| **PATCH** | 笔误修正、补充遗漏评审项 | v1.0.0 → v1.0.1 |

**多轮评审规则：** 被评审文档修改后重新评审时，**新建文件**并在文件名加 `_R2`、`_R3` 后缀，版本重置为 `v1.0.0`。

**文件命名规范：**
```
doc/review/{项目名}_需求评审报告{_RN}.md
doc/review/{项目名}_架构评审报告{_RN}.md
doc/review/{模块名}_详设评审报告{_RN}.md
doc/review/{模块名}_测试用例评审报告{_RN}.md
```
